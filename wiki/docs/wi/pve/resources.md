Отдельного глобального параметра «коэффициент переподписки CPU/RAM», который можно изменить одной настройкой для всего гипервизора нет.
Коэффициент переподписки формируется фактически, через суммарное количество ресурсов, назначенных виртуальным машинам.

Для CPU - Коэффициент CPU-переподписки = сумма vCPU всех ВМ / количество физических CPU-ядер/потоков узла

Если на узле доступно 32 логических CPU, а всем ВМ суммарно выделено 64 vCPU, коэффициент переподписки будет 1:2.
Изменяется он не отдельной настройкой гипервизора, а через параметры конкретных ВМ:

- количество sockets/cores/vCPU у ВМ;
- лимиты CPU, если они используются;
- распределение ВМ по узлам кластера.

Для RAM аналогично - Коэффициент переподписки памяти = сумма назначенной ВМ памяти / физический объём RAM узла
Изменяется он через параметры памяти конкретных ВМ:

- объём Memory;
- Ballooning, если используется;
- лимиты/резервирование памяти;
-  распределение ВМ по узлам.

То есть практическая инструкция по изменению коэффициента такая:

1. Посчитать физические ресурсы узла: количество CPU и объём RAM.
2. Посчитать суммарно выделенные ВМ vCPU и RAM.
3. При необходимости изменить параметры ВМ, уменьшить/увеличить vCPU и RAM.
4. Перераспределить ВМ между узлами кластера, если требуется снизить нагрузку на конкретный узел.
5. Оставить резерв ресурсов для самого гипервизора и системных сервисов.

Не рекомендуется стремиться к 100% использованию всех ресурсов хоста виртуальными машинами, особенно по памяти. Для стабильной работы необходимо оставлять запас под сам гипервизор, службы хранения, сеть, процессы QEMU/KVM и возможные пики нагрузки.
