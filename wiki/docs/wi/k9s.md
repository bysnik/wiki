https://k9scli.io/

## Установка

```bash
apt-get install k9s
```

### Предполетная проверка

K9s использует терминальный режим с 256 цветами. Убедитесь, что параметр TERM настроен соответствующим образом.
```bash
export TERM=xterm-256color
```

Чтобы выдавать команды редактирования ресурсов, убедитесь, что переменные окружения EDITOR и/или KUBE_EDITOR установлены.
```bash
# Kubectl edit command will use this env var.
export KUBE_EDITOR=my_fav_editor
```

## Команды K9s

### Аргументы командной строки

Интерфейс K9s поддерживает ряд аргументов, позволяющих запускать инструмент с различными настройками.

```bash
# Показать все доступные опции CLI
k9s help

# Получить информацию о среде выполнения K9s (логи, конфигурации и т.д.)
k9s info

# Запустить K9s в указанном пространстве имён (namespace)
k9s -n mycoolns

# Запустить K9s и сразу открыть представление Pod'ов
k9s -c pod

# Запустить K9s с контекстом, отличным от контекста по умолчанию в KubeConfig
k9s --context coolCtx

# Запустить K9s в режиме «только чтение» — все команды изменения отключены
k9s --readonly
```

## Горячие клавиши

| Действие | Команда | Примечание |
|----------|---------|------------|
| Показать активные клавиатурные подсказки и справку | `?` | |
| Показать все доступные псевдонимы ресурсов | `ctrl-a` | |
| Выйти из K9s | `:q`, `ctrl-c` | |
| Просмотр ресурса Kubernetes (в единственном/множественном числе или по короткому имени) | `:pod⏎` | Принимает единственное, множественное число, короткое имя или псевдоним, например `pod` или `pods` |
| Просмотр ресурса в определённом пространстве имён | `:pod ns-x⏎` | |
| Просмотр Pod'ов с фильтром | `:pod /fred⏎` | Показывает все Pod'ы, отфильтрованные по строке `fred` |
| Просмотр Pod'ов по меткам (labels) | `:pod app=fred,env=dev⏎` | Показывает все Pod'ы с метками `app=fred` и `env=dev` |
| Просмотр Pod'ов в заданном контексте | `:pod @ctx1⏎` | Показывает все Pod'ы в контексте `ctx1`. Переключает текущий контекст K9s! |
| Фильтрация представления ресурсов | `/filter⏎` | Поддерживает регулярные выражения, например `fred\|blee` для фильтрации ресурсов с именами `fred` или `blee` |
| Инверсный фильтр по регулярному выражению | `/! filter⏎` | Оставляет всё, что **не** соответствует фильтру |
| Фильтрация представления ресурсов по меткам | `/-l label-selector⏎` | |
| Нечёткий поиск ресурса по фильтру | `/-f filter⏎` | |
| Выход из режима просмотра / команды / фильтра | `<esc>` | |
| Описание, просмотр, редактирование, просмотр логов и т.д. | `d`, `v`, `e`, `l` … | |
| Переключиться на другой контекст Kubernetes (в представлении Pod'ов) | `:ctx⏎` | |
| Переключиться на другой контекст Kubernetes напрямую (последнее использованное представление) | `:ctx context-name⏎` | |
| Переключиться на другое пространство имён Kubernetes | `:ns⏎` | |
| Просмотр всех сохранённых ресурсов (снимков экрана) | `:screendump` или `sd⏎` | |
| Удалить ресурс (подтверждение с помощью `TAB` и `ENTER`) | `ctrl-d` | |
| «Убить» ресурс (без диалога подтверждения, аналогично `kubectl delete --now`) | `ctrl-k` | |
| Открыть представление «Пульс» (Pulses) | `:pulses` или `pu⏎` | |
| Открыть представление «Рентген» (XRay) | `:xray РЕСУРС [ПРОСТРАНСТВО-ИМЁН]⏎` | `РЕСУРС` может быть одним из: `po`, `svc`, `dp`, `rs`, `sts`, `ds`; `ПРОСТРАНСТВО-ИМЁН` — опционально |
